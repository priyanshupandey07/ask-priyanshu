## 🧠 First: what is RAG? (60-second version)

A language model only "knows" what it was trained on, and it sometimes makes things up. **RAG fixes that** by giving the model the right facts *at question time*:

1. **Retrieve** — find the most relevant pieces of *your* documents for the question.
2. **Augment** — paste those pieces into the prompt as context.
3. **Generate** — the model answers using only that context.

> Analogy: it's an open-book exam. Instead of memorising everything, you look up the relevant page first, then answer from it.

That's the whole idea. This project does exactly those three steps over `profile.md`.

## 🚀 Run it in 5 steps (total beginner friendly)

### Step 0 — Install Python (one time)
Download Python 3.10+ from python.org. During install on Windows, **tick "Add Python to PATH"**.

### Step 1 — Open a terminal in this folder
- Windows: open the folder, type `cmd` in the address bar, press Enter.
- Mac: right-click the folder → "New Terminal at Folder".

### Step 2 — Install the libraries
```bash
pip install -r requirements.txt
```
(First time, `sentence-transformers` downloads a model — a few hundred MB. Be patient. If it fails or is too big, the app still works using the lighter keyword search automatically.)

### Step 3 — Try the engine in the terminal (no website yet)
```bash
python rag.py "What are Priyanshu's strongest skills?"
```
You'll see the answer and the facts it used. **This already works with no API key and no cost.**

### Step 4 — Launch the website
```bash
streamlit run app.py
```
Your browser opens a chat page. Ask questions. Done — you've built a RAG app! 🎉

### Step 5 — (Optional) Nicer answers with a FREE LLM key
Without a key, the app shows the most relevant facts ("extractive" mode). To get smooth, natural sentences, add a **free Groq key**:
1. Sign up at console.groq.com and create an API key (free).
2. Set it before running:
   - Windows: `set GROQ_API_KEY=your_key_here`
   - Mac/Linux: `export GROQ_API_KEY=your_key_here`
3. Run again — answers are now generated by an LLM, grounded in your facts.

(Prefer OpenAI? Set `OPENAI_API_KEY` instead; the code supports both.)

## ✍️ Make it yours
- **Change what it knows:** edit `profile.md` (add a paragraph, save, re-run). No code changes needed.
- **Retrieve more/fewer facts:** change `TOP_K` in `rag.py`.
- **Swap the model:** set the `LLM_MODEL` environment variable.
